Lab Overview

#overview

The Architecture Design Lab is the FERROS-native workspace for building design. Users create and manage architectural drawings through a structured hierarchy—from workspace down to individual annotations—using sheetgen's CQRS command bus and botgen's agent orchestration pipeline.

AI agents assist with drafting, rendering, and analysis, but every AI action requires explicit user approval. The consent-first model ensures no automated drawing generation, rendering, or modification happens without your confirmation.

Backend: Axum REST API · CQRS via POST /command/{name} · definitions.yaml as authoritative command schema · PostgreSQL + SQLx persistence · Tigris asset storage.

Drawing Hierarchy Explorer

#hierarchy
Sheetgen Domain Stack

The sheetgen domain model is a strict top-down hierarchy. Each level contains the next, enforcing structured ownership and cropping boundaries throughout the drawing pipeline.

📁 Workspace
Top-level organizational container. Groups related projects under a single user or team scope.
Root
📋 Project
A building design project. Contains all sheets, drawings, and configuration for a single building or site.
Container
📄 Sheet
A printable drawing sheet with defined size, scale, and layout. Contains titleblocks and annotations.
Template ✓
🏷️ Titleblock
Border and information block that frames the sheet. Crops viewports to its margins. Contains project metadata.
Template ✓Crops → Viewport
🔲 Viewport
A rectangular window into a view. Defines the visible region and scale factor. Crops the view it contains.
Template ✓Crops → View
👁️ View
A named perspective of the model (floor plan, elevation, section, detail). Crops drawings to its boundary.
Template ✓Crops → Drawing
✏️ Drawing
The actual architectural drawing content — geometry, elements, layers. The base content that views display.
Template ✓Content
📌 Annotation
Labels, dimensions, notes, symbols. Attaches at any level with single-parent constraint. Cropped by parent scope.

Rendering Pipeline

#rendering

The rendering pipeline is powered by botgen's Rendering Service, which compiles drawing hierarchies into exportable output through schema-driven generation of floor plans, elevations, sections, and details.

Render Flow
1
Input — Domain Data Collection
Gather drawing elements, style definitions, target resolution, and output format from the sheetgen domain model. Resolve all parent-child relationships and compute effective transforms.
Elements Style Resolution Format
2
Layer Compilation
Flatten the drawing hierarchy into ordered layers. Resolve z-ordering, visibility flags, and layer-level style overrides. Each drawing becomes a compositable layer in the render stack.
Layer Stack Z-Order Visibility
3
Cropping Hierarchy Enforcement
Apply the cropping chain: titleblocks crop viewports, viewports crop views, views crop drawings. Each boundary is computed and clipping masks are generated for the render compositor.
Clip Masks Boundary Calc Containment
4
Annotation Overlay
Render annotations at each hierarchy level. Sheet annotations overlay everything. Lower-level annotations are clipped to their parent scope. Text, dimensions, and symbols are rasterized with font and style settings.
Labels Dimensions Symbols
5
Output — Export
Composite all layers and annotations into the final output. Export to the requested format. Assets are stored in Tigris with metadata linking back to the source drawing hierarchy.
PNG SVG PDF

Cropping & Layer System

#cropping

The cropping hierarchy enforces strict visual containment at every level. Annotations at each level are cropped to their parent scope, ensuring that drawing content never visually leaks outside its designated boundary.

Visual Containment Hierarchy
📄 Sheet Top Level
Sheet annotations overlay everything — they are never cropped. Title text, sheet numbers, and global notes render above all other content.
🏷️ Titleblock Margin Boundary
Titleblock annotations respect margins. Border lines, firm info, and revision blocks are constrained to the titleblock region.
🔲 Viewport Viewport Crop
Viewport annotations are cropped by the viewport boundary. Scale labels, viewport titles, and frame lines clip at the viewport edge.
👁️ View View Crop
View annotations are cropped by the view boundary. Section marks, elevation indicators, and callout bubbles clip at the view edge.
✏️ Drawing Content
Drawing annotations are content-specific. Dimensions, material hatches, and element labels are bound to the drawing geometry they describe.

Archifusion Workshop Mode

#workshop

The Archifusion Workshop is an educational mode from botgen where students learn building design through hands-on interaction with an AI Architect NPC. Every AI-powered action requires explicit consent — students maintain full control over their creative process.

Workshop Flow
1
Student Joins
Student enters the workshop via Discord. A personal channel is created with isolated workspace.
2
Architect AI NPC Deployed
A SheetgenAgent instance is spawned as the student's personal Architect NPC, ready to assist with design guidance.
3
Image Analysis
Student uploads hand-drawn sketches. The LLM Client analyzes the image and suggests sheetgen commands to digitize the design.
4
3D Model Visualization
Digitized drawings are compiled into a 3D model preview. Students see their building take shape from multiple views.
5
Build Guide Generation
Sheetgen generates a complete build guide with annotated drawings, material lists, and construction sequences.

AI Action Consent Gates

Allow Architect AI to analyze your uploaded sketch and suggest drawing commands.
Allow Architect AI to create a floor plan drawing from your sketch analysis.
Allow Rendering Service to generate a 3D model preview from your drawings.
Allow Architect AI to generate a build guide with material estimates for your project.
Allow Architect AI to add structural annotations to your elevation drawing.
Allow Rendering Service to export your complete sheet set as a PDF package.

Risk & Constraints

#risks
Risk Source Severity Mitigation Strategy
Rendering complexity at scale Sheetgen HIGH Progressive rendering, resolution tiers, async export pipeline with queue management
Template propagation in command layer Sheetgen HIGH Comprehensive integration tests, command-level validation, audit trail for all template operations
Cropping hierarchy edge cases Sheetgen MEDIUM Property-based testing for boundary conditions, visual regression tests for render output
Platform coupling (Discord dependency) Botgen HIGH Gateway trait abstraction, MessageBus interface decouples agent logic from transport
Service mesh migration complexity Botgen MEDIUM Trait adapters for all services, incremental migration with dual-mode operation
Agent autonomy safety Botgen HIGH Consent-first model, all AI actions require explicit user approval, moderation middleware
LLM provider reliability Botgen MEDIUM Provider-agnostic LLM trait, fallback chains, graceful degradation without AI features
Multi-step undo consistency Sheetgen MEDIUM Event sourcing pattern, immutable command log, scope-isolated undo stacks
Asset storage costs Sheetgen LOW Tiered storage, lazy rendering, cache invalidation on domain changes
Workshop scalability (concurrent students) Botgen MEDIUM Agent pooling, resource limits per session, Redis-backed session state in M8
