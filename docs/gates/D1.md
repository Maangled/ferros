# Gate D1 — Demo

**Status:** 🟡 Active (defined, not closed)
**Stream:** S7 primary; S5 consumer awareness
**Sits between:** G3 (closed) and G4 (Launch)
**Unlocks:** operator-attended demo readiness; basis for partner-facing demonstration planning

---

## What D1 is

D1 is a **demo gate**, not a launch gate. It certifies that a single device, under operator attendance, can show a complete but not production-hardened FERROS flow end to end. D1 is the milestone at which the project can run a credible live demonstration to a partner or small audience without requiring the full G4 hardening criteria.

Partner-facing or public communications continue to use G4 wording. D1 is internal evidence that the system is demonstrable.

---

## Entry criteria

- G3 is closed.
- A primary D1 device target has been selected and documented in `docs/hardware/d1-target-inventory.md`.
- The FERROS binary builds and runs on the target device (or an equivalent x86_64 stand-in for the first proof run).

---

## Required evidence

All of the following must be true simultaneously. Evidence must be documented in this file under the Evidence section before D1 is declared closed.

- [ ] **Profile creation / show:** `ferros profile init` and `ferros profile show` both run successfully on the target device using the real FERROS binary and the frozen `profile.v0.json` contract. Profile persists across a shell re-open within the same session.
- [ ] **Home Assistant entity registered:** at least one Home Assistant entity is registered through the planned bridge contract **or a documented stand-in** (the stand-in must be named in the evidence; a pure mock without a defined bridge seam does not count). Operator-attended is acceptable.
- [ ] **Consent flow visible:** the consent/capability grant flow is visible to the operator on the localhost shell or equivalent operator surface. Deny-by-default enforcement must be demonstrable (one denied request logged and visible).
- [ ] **Reboot-safe FERROS-side state:** the device goes through one full power cycle. After reboot, the FERROS profile loads and at least one agent re-registers without manual intervention. HA re-registration is not required for D1 (it is required for G4).
- [ ] **Operator attended throughout.** Unattended operation is NOT required for D1.

---

## What D1 explicitly does NOT require

D1 does not require (and must not claim) any of the following:

- Hardened HA fork integration
- Multi-device operation
- Unattended or production-grade reliability
- An independent install (required for G4)
- `docs/hub/reference-hardware.md` finalised (encouraged but not mandatory for D1)
- `LAUNCH.md` updated with a confirmed install date (required for G4)

---

## Exit criteria

All required evidence items checked and documented in this file. The demo is declared D1-ready in `STATUS.md` and `WAVE-RUN-LOG.md`. No version tag is required for D1 (D1 is a milestone, not a release).

---

## Relationship to G4

G4 (Launch) criteria are unchanged. D1 evidence does not substitute for G4 evidence. The evidence tables are separate. Completing D1 advances the project toward G4 but does not constitute any G4 item being checked.
