# Gate G4 — Launch

**Status:** 🟡 Active  
**Stream:** S7  
**Unlocks:** Public announcement (optional, after private beta); `v0.2.0` tag

---

## Entry criteria

- G3 is closed (agent center on runtime demo; `ferros-node demo` green in CI).
- S7 implementation work is underway.
- At least one home-server or Raspberry Pi is available for testing.

---

## Required evidence

All of the following must be true simultaneously. Evidence must be documented in this file under the Evidence section before G4 is declared closed.

- [ ] `ferros-hub` binary cross-compiles for `aarch64-unknown-linux-gnu` (or `x86_64-unknown-linux-gnu` for home server).
- [ ] `ferros-hub` runs on a physical device (not CI, not QEMU, not developer laptop).
- [ ] Device profile created via `ferros profile init` on the target device; persists after restart.
- [ ] `ferros agent list` shows at least one registered HA-bridge agent on the running hub.
- [ ] At least one real Home Assistant entity is registered through the FERROS agent center — not mocked, not stubbed.
- [ ] The HA entity is visible in the HA dashboard and reflects real state.
- [ ] Consent enforced: a capability request that was not granted is denied at the bus level, logged, and the denial is visible in the HA UI or `ferros agent logs`.
- [ ] Hub survives a full power cycle: after reboot, profile loads, agents re-register, HA entity is restored without manual intervention.
- [ ] At least one independent private-beta install confirmed (a separate home setup, not the primary developer's machine).
- [ ] `docs/hub/reference-hardware.md` documents the tested hardware.
- [ ] `LAUNCH.md` is updated with the confirmed install date and hardware specification.

---

## Exit criteria

All required evidence items checked and documented in this file. `v0.2.0` tagged. Private beta announced to ≤5 trusted users. No wider public announcement until private beta feedback is gathered (2–4 weeks minimum).
