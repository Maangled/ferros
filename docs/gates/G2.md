## Notes

G2 is closed. The freeze line and the final real-binary CLI lifecycle proof are both landed, so downstream streams can treat the S2 v0 profile and grant boundary as stable. G3 is now the active gate.

The lightweight profile type surface is still worth distinguishing from the frozen consumer boundary: `ProfileId` and `ConsentManifest` are already public in `ferros-profile`, but G2 only closed once the broader frozen profile fixture, freeze boundary, and full CLI lifecycle evidence landed alongside them.

The signed `CapabilityGrant` envelope path is now landed in `ferros-profile`: Ed25519 sign and verify are implemented, the stripped JSON payload contract is explicitly frozen in `schemas/capability-grant.v0.json`, valid plus invalid grant fixtures are covered by `cargo test -p ferros-profile`, and the H1 contract validator now embeds `schemas/capability-grant.v0.json`, validates `grant-valid.json` against it, and classifies invalid signature rejection as semantic rather than schema-level.

Before closeout, the convergence path already exercised the real current `CapabilityGrant` type through `ferros-node demo` and the local `ferros agent ...` CLI. Profile-side evidence is no longer limited to fixture parsing: `ferros-profile` now owns fresh Stage 0 profile creation plus create-without-overwrite filesystem persistence, the dedicated `schemas/fixtures/profile-valid.json` golden fixture is proven against the frozen `schemas/profile.v0.json` unsigned contract in both `ferros-profile` and H1, and `ferros-node` wires a minimal `ferros profile init [path]` / `ferros profile show [path]` slice on top of that boundary with focused cargo-test coverage.

Key material and full profile signing evidence are now in the crate as well: `KeyPair` generates Ed25519 key material with a required local device label, derives `ProfileId` from the verifying key, and signs the additive `SignedProfileDocument` envelope so `cargo test -p ferros-profile` now proves create → serialize → sign → verify → revoke without widening `schemas/profile.v0.json`. The freeze boundary is now explicit: `schemas/profile.v0.json` is the frozen unsigned published v0 contract, while `SignedProfileDocument` stays Rust-local at v0 and the signed-profile fixture revalidates its embedded `profile` payload against that frozen contract rather than a published signed-profile schema. The final CLI closeout slice now carries that boundary through the real `ferros` binary: `cargo test -p ferros-node` launches `ferros profile init`, `grant`, `export`, `import`, `revoke`, and `show` against temp-file-backed local state, confirms `show` still emits the unsigned `ProfileDocument`, and verifies the persisted revoked grant remains within the frozen `schemas/capability-grant.v0.json` envelope shape. With that landed evidence, G2 is closed.
