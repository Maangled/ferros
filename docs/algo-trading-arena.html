Technical Architecture

All data local-first, encrypted at rest, no cloud required, AI runs locally when possible. Smart contract boundaries follow the ADR-002 pattern. Click any layer to expand details.

Core Trading Stack
🔑 FERROS Kernel
Memory-safe foundation, capability-based security
RustCore
The FERROS microkernel provides the security foundation. All trading operations run in isolated capability-scoped processes.
✅ Consent Engine
Trade Window pattern — every action requires explicit approval
Security
Implements the Trade Window consent model. No trade executes without user confirmation. Agents propose, humans approve.
📊 Trading Engine
Order management, position tracking, P&L calculation
EngineReal-time
Manages all trading operations: order placement, position lifecycle, profit/loss tracking, and portfolio state.
🃏 Card System
Asset → Card mapping, abilities, rarity, evolution, deck management
Gamification
Maps financial assets to trading cards. Manages rarity tiers, keyword abilities, card evolution, and deck composition rules.
🖥️ UI Layer
Self-contained HTML/CSS/JS, game board, card rendering
Frontendfile://
Single-file HTML prototype. Zero external dependencies. Works on file:// protocol. All CSS and JS inline.
AI Agent Pipeline
🤖 AI Agent Host
Agent lifecycle, capability grants, trust scoring
Agent
Hosts Scout, Strategist, Guardian, Oracle, and Broker agents. Manages trust levels and capability boundaries.
🧠 Strategy Engine
Deck recommendations, market analysis, pattern recognition
AI
Runs local ML models for pattern analysis. Provides deck optimization and trade recommendations.
🛡️ Risk Manager
Exposure limits, stop-loss automation, diversification enforcement
Safety
Guardian agent backend. Enforces user-defined risk boundaries. Can veto proposed trades that exceed limits.
⚙️ Execution Bridge
Order routing, fill management, timing optimization
Exec
Bridges consent-approved trades to execution. Optimizes entry/exit timing within user-defined parameters.
Data & Progression Layer
🔗 Seal Chain
Tamper-evident trade history, SHA-256 with djb2 fallback
Crypto
Every trade sealed into an append-only chain. SHA-256 primary with djb2 fallback for file:// protocol compatibility.
🎟️ NFT Minting Bridge
Consent-gated on-chain ticket generation
Optional
Optional bridge to mint sealed trades as on-chain NFT tickets. Fully consent-gated. Open source minting code.
🎮 Loot & Idle Engine
Reward calculator, idle progression, prestige system
GameXP
Tracks loot box triggers, idle earnings, prestige multipliers, and creature evolution milestones. All progression client-side.
