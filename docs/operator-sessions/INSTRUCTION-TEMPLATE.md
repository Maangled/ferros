# Operator Instruction Template

Use this template when a Human Test Backlog item moves to `Ready`.

```text
Session ID:
Date:
Operator:
Backlog item:

Goal:

Commands or routes to use:

Expected observation:

Evidence capture path:

Rollback path:

Stop criteria:

Immediate-task comments:

Meta comments for coordinator:
```

Notes:

1. Use one instruction packet per session window unless several items are intentionally grouped.
2. Keep commands/routes exact.
3. Name whether the expected result is proposed, blocked, rehearsal, runway, or canonical.
4. If the session exposes a blocking local defect, mark it clearly so the coordinator can route a hotfix.
